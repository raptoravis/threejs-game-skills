# Prompt Templates (Phaser Game Director)

Load this reference only when the user asks for reusable prompts, starter prompts, or a task template.

## Build a Premium 2D Game

```text
Build a premium 2D <genre> game using Phaser 4 + TypeScript + Vite + Matter.js.

Core loop: <one-sentence verb + objective + feedback + fail/retry>

Follow the Phaser game director phase OS:
1. Discovery: scaffold project, state the loop, initialize ledgers.
2. Gameplay: implement playable loop with scenes, input, entities, Matter.js physics, HUD, audio hooks.
3. Asset sourcing: run credential probe, decide sprite/audio sources.
4. 2D graphics: score visuals, upgrade sprites/tilemaps/particles/palette.
5. UI: polish HUD, menus, responsive fit, touch targets.
6. Debug/profile: fix issues, measure performance.
7. QA/release: build, screenshot, canvas pixels, release notes.

Quality bar: premium (2D visual scorecard all categories >=2, average >=2.3).

Verify with build, browser, console check, desktop + mobile screenshots, canvas pixel check, and real input path.
```

## Upgrade 2D Game Visuals

```text
Upgrade this Phaser 2D game's visuals to premium quality.

1. Capture current active-play screenshots (desktop + mobile).
2. Score all 10 categories of the 2D visual scorecard.
3. Identify 3 lowest-scoring categories.
4. Upgrade: spritesheets, tilemaps, parallax, particles, color palette, 2D lighting.
5. Re-capture screenshots, re-score, verify improvement.
6. Report before/after scorecard.

Quality bar: every category >=2, average >=2.3.
```

## Debug and Finish 2D Game

```text
Debug and finish this Phaser 2D game.

1. Check console for errors and warnings.
2. Test main input path: can player complete the core loop?
3. Test scene transitions: no flicker, no stale objects.
4. Test restart: all physics bodies, listeners, tweens cleaned up.
5. Profile: body count, sprite count, frame time at peak gameplay.
6. Mobile: test touch controls, viewport scaling, safe areas.
7. QA: build, screenshots, canvas pixel check.
8. Report issues found and fixes applied.
```
